# Plan — Add a new vertical resource

This is the execution guide for adding (or completing) a **vertical resource** in ECP: a
self-contained slice that cuts through `domain → backend (CR + controller + plugin) →
frontend (REST)`, plus the dummy CSP plugin, deployment/RBAC, tests, and docs.

It is a **checklist, not a script**. Follow the steps in order; explore the canonical
references when a detail is unclear. The two complete verticals are the ground truth:

- **`resource/workspace/v1/`** — regional-tenant-scoped reference.
- **`resource/storage/block-storage/v1/`** — regional-workspace-scoped reference (richest:
  has an extra `IncreaseSize` operation and a multi-state reconciler).

Build the new resource by copying these patterns and adapting them to the new resource's
spec, scope, and API group.

---

## 0. Hard rules (read first)

1. **The SECA spec is the source of truth.** The resource name, fields, requirements, and
   validations come from the spec, vendored at [modules/go-sdk/spec/](../../../modules/go-sdk/spec/).
   The new resource must strictly abide by it.
2. **Never edit the submodules** `modules/go-sdk/` or `modules/go-sdk/spec/`. CI fails if
   they are modified by hand. They are read-only inputs to code generation.
3. **Follow [doc/CONVENTIONS.md](../../../doc/CONVENTIONS.md)** for every hand-written line
   (see §6). Generated `zz_generated_*` files are exempt and never hand-edited.
4. **Do not auto-commit.** If a commit is truly necessary, create a *separate* branch and
   use a one-line [Conventional Commit](https://www.conventionalcommits.org/) message — no
   "generated by an agent" attribution, no co-author trailer.
5. **Verify only at the very end** with `make pre-commit-ctzd` then `make pre-merge-ctzd`
   (§7). Their output is long; running them mid-way wastes time. **Docker is the only
   required toolchain.** If Docker (or the toolchain) is unavailable, **warn the user and
   mark the work UNVERIFIED** — do not claim it passes.
6. **Static analysis alone will not fully explain code generation.** Generators are wired
   loosely (one invocation — `conditioned-gen` — has *no* visible Makefile/`go:generate`
   hook at all). When in doubt, **run the generation step and observe which files change**
   rather than trying to predict it.

## 1. Preflight — resolve identity and compute the delta

Do this before writing any code. A resource may already be partially scaffolded; you only
do the **missing** steps.

1. **Resolve the canonical name from the spec.** Open the group file
   `modules/go-sdk/spec/spec/resources/<group>.v1.yaml` and find the resource's top-level
   `name:` (singular) and `plural:`. Use these *exact* strings everywhere (dir name, CRD
   path, RBAC). Examples: storage `image` → plural `images`; network `network` →
   `networks`. ⚠️ User-facing aliases may not match the spec — e.g. there is **no `vpc`**
   resource; the SECA "VPC" is the resource named **`network`**. If you can't find the
   name in the spec, stop and ask — do not invent one.

2. **Confirm the generated Go schema exists.** The slice's K8s types are generated from
   `modules/go-sdk/pkg/spec/schema/<resource>.go`, which must already define
   `<Kind>Spec` / `<Kind>Status`. If that file is missing, the resource is not yet in the
   pinned go-sdk version. **STOP**: the spec is upstream-owned and the submodule is
   read-only — request a go-sdk version bump; do not hand-author the schema.

3. **Classify the scope from the spec `hierarchy:` field.** This single field drives the
   metadata embed, the kubebuilder scope, the REST path depth, and which gateway serves it:

   | Scope | spec `hierarchy` | examples | domain embed | kubebuilder | REST path | gateway |
   |---|---|---|---|---|---|---|
   | regional-tenant-scoped | `[tenant]` | workspace, **image**, *-sku | `domain.RegionalMetadata` (Tenant only) | `scope=Namespaced` | `/v1/tenants/{tenant}/<plural>` | regional |
   | regional-workspace-scoped | `[tenant, workspace]` | **block-storage**, network | `domain.RegionalMetadata` (Tenant+Workspace) | `scope=Namespaced` | `/v1/tenants/{tenant}/workspaces/{workspace}/<plural>` | regional |
   | network-scoped (deeper) | `[tenant, workspace, network]` | subnet, route-table | `domain.RegionalMetadata` + parent ref | `scope=Namespaced` | `.../networks/{network}/<plural>` | regional |
   | global / cluster-scoped | *(none)* | region | `domain.Metadata` (no scope) | `scope=Cluster` | `/v1/<plural>` | global |

4. **Identify the API group and its REST handler owner.** REST is **per-API-group, not
   per-resource**. One slice in a group owns `frontend/rest/handler.go` implementing the
   *whole* group's `ServerInterface`; sibling slices contribute only
   `frontend/rest/converter.go`. For the storage group the owner is **block-storage**
   ([resource/storage/block-storage/v1/frontend/rest/handler.go](../../../resource/storage/block-storage/v1/frontend/rest/handler.go));
   for network it is **network** ([resource/network/network/v1/frontend/rest/handler.go](../../../resource/network/network/v1/frontend/rest/handler.go)).
   Decide: is your resource a **new resource in an existing group** (extend the owner's
   handler + add your converter) or a **new group** (create a new handler + register a new
   gateway block at a new `BaseURL`)?

5. **Assess current state and compute the delta.** List the slice dir and compare to a
   complete vertical. Typical states:
   - **Bare scaffold** (e.g. `resource/storage/image/v1/` today): only
     `backend/kubernetes/{resource.go, generate.go}` + `zz_generated_*`. Everything else is
     missing — do the full build.
   - **Partial**: some of `domain.go` / `conversion.go` / `plugin*.go` present. Fill gaps.
   - **Complete** (e.g. `resource/network/network/v1/` today): has `domain.go`, all of
     `backend/kubernetes/{conversion,plugin,plugin_handler,controller}.go`, and
     `frontend/rest/{handler|converter}.go`. **Nothing to do** for the slice — report it as
     already implemented.

   ⚠️ A code-complete slice may still be **incompletely wired**: `network` is fully
   implemented and registered in `main.go`, yet its `networks` rule is **missing from the
   dummy delegator ClusterRole**. So even when the slice looks done, separately verify the
   downstream wiring — gateway registration, dummy `main.go`, RBAC across *all* ClusterRoles
   (§4.13), and tests — and fill any gaps.

   When unsure whether a generated file is current, regenerate (§3) and observe — do not
   assume from a stale tree.

## 2. Repository map (so you don't have to re-explore)

- **`resource/<group>/<resource>/v1/`** — the vertical slice. Holds `domain.go` (canonical
  type + identity constants), `backend/kubernetes/` (CR type, conversion, controller,
  plugin interface + handler, generated files), and `frontend/rest/` (REST↔domain
  converters and, for the group owner, HTTP handlers).
  ⚠️ **Region path caveat:** `region` lives at **`resource/region/v1/`** — *not*
  `resource/<group>/region/v1/`. It uses the bare group `v1.secapi.cloud`, is
  `scope=Cluster`, has **no plugin/controller**, and is served by the **global** gateway.
  Most resources follow the `resource/<group>/<resource>/v1/` shape instead.
- **`resource/common/`** — shared domain types (`Reference`, `Status`, `ResourceState`,
  `RegionalMetadata`/`Metadata`) and shared `FromCR`/`ToCR` helpers (`commonbackend`). Reuse
  these; never re-implement state conversion.
- **`framework/`** — resource-agnostic SDK. `framework/backend/kubernetes/` holds the
  adapters, the generic controller/plugin-handler, the codegen tools (`cmd/model-gen`,
  `cmd/inject-kubebuilder-markers`, `cmd/conditioned-gen`), and **the Makefile with the
  hardcoded slice list** (§3). `framework ↛ resource` is compiler-enforced.
- **`gateway/cmd/`** — the API-server binaries. `regionalapiserver.go` wires tenant/
  workspace/network-scoped handlers; `globalapiserver.go` wires the global (region) handler.
- **`chart/crd/`** — generated CRD YAML (`<group>.v1.secapi.cloud_<plural>.yaml`). Output
  only; produced by generation.
- **`csp/dummy/`** — the reference CSP plugin (mock backend). `pkg/plugin/` has one file per
  resource, `cmd/main.go` wires them, `deploy/` holds manifests + RBAC, `test/integration/`
  holds plugin integration tests. **Only extend `dummy`** — leave `ionos`/`aruba` alone.
- **`test/e2e/`** — full-stack harness: `test/integration/{delegator,gateway-global,
  gateway-regional}/` tests and `deploy/{delegator,gateway-regional,gateway-global,
  test-data}/` manifests.

## 3. Code generation — how spec becomes types + CRDs (and the silent trap)

Pipeline: `spec yaml → (oapi-codegen, upstream) → modules/go-sdk/pkg/spec/schema/<resource>.go
→ (model-gen, per slice) → zz_generated_schema.go → (inject-kubebuilder-markers) → markers
→ (controller-gen crd) → chart/crd/<group>.v1.secapi.cloud_<plural>.yaml`.

Two entry points, **both required** for a new slice:

- `(cd resource && go generate ./...)` — runs each slice's `//go:generate` (model-gen for
  `zz_generated_schema.go`; mockgen for the test mocks). Driven by `generate.go`.
- `make generate-api` — runs the shared schema types **and** CRD generation: marker
  injection over the slice list, then `controller-gen crd`.

> ### ⚠️ The hardcoded slice list (the one Makefile edit you must make)
> `controller-gen crd` auto-discovers your CRD via the glob `paths="…/resource/..."`, **but
> kubebuilder-marker injection runs off a hardcoded list** in
> [framework/backend/kubernetes/Makefile](../../../framework/backend/kubernetes/Makefile)
> (the `for dir in … done` loop in `generate-crds`). **Add your slice's
> `backend/kubernetes` dir to that loop.** If you omit it, the CRD still ships — but
> **silently without its spec validations** (required/enum/min/max/CEL), and the
> `generate-api-verify` gate stays green because the tree is self-consistent. This is the
> single most common silent failure. It is the *only* Makefile edit needed; no other wiring.

> ### Don't try to trace generation statically
> The `conditioned-gen` tool (which emits `zz_generated.conditions.go` for
> `+ecp:conditioned` CR types) has **no discoverable invocation** in any Makefile, script,
> or `go:generate` directive. Marker→CRD lowering by controller-gen is likewise only
> observable by running it. After generating, **inspect the actual outputs**: confirm
> `zz_generated_schema.go`, `zz_generated.conditions.go`, `zz_generated.deepcopy.go`, and
> `chart/crd/<group>.v1.secapi.cloud_<plural>.yaml` exist and that the CRD contains your
> spec's validations. If the conditions file is stale, run the tool at
> `framework/backend/kubernetes/cmd/conditioned-gen` over the slice.

Reference: [doc/CODEGEN.md](../../../doc/CODEGEN.md).

## 4. Build steps (dependency order)

Each step: the file to create, its canonical reference, and what is scope/group-specific.
`<X>` = Kind (e.g. `Image`), `<x>dom`/`<x>k8s` = the import aliases, `<plural>` from the spec.

> **Types → CRD → controller → plugin → REST → wiring → tests → docs.**

### 4.1 Domain type — `resource/<group>/<resource>/v1/domain.go`
Canonical domain struct + identity constants (`Kind`, `Resource`, `Group`, `Version`,
`ProviderID`). Embed `domain.RegionalMetadata` (tenant/workspace-scoped) or `domain.Metadata`
(global). Define `<X>Spec` / `<X>Status` (status embeds `domain.Status`).
Ref: [resource/storage/block-storage/v1/domain.go](../../../resource/storage/block-storage/v1/domain.go).

### 4.2 CR wrapper — `resource/<group>/<resource>/v1/backend/kubernetes/resource.go`
**This is the file the spec's requirements/validations land in** (as kubebuilder markers —
mostly auto-injected from go-sdk struct tags). Define group/version constants, `<X>GVR`,
`<X>GVK`, the `SchemeBuilder`, the CR struct (`TypeMeta`+`ObjectMeta`+`Spec`+
`schemav1.CommonData`+`*Status`), the `<X>List`, and `init()` registration. Set the
kubebuilder markers, **including the correct `scope=`** and `+ecp:conditioned` if the
resource has status conditions.
Ref: [resource/storage/block-storage/v1/backend/kubernetes/resource.go](../../../resource/storage/block-storage/v1/backend/kubernetes/resource.go).
(A bare scaffold like [resource/storage/image/v1/backend/kubernetes/resource.go](../../../resource/storage/image/v1/backend/kubernetes/resource.go)
already has this — verify it, don't recreate it.)

### 4.3 Generate directives — `resource/<group>/<resource>/v1/backend/kubernetes/generate.go`
`//go:generate` for `model-gen` (`--root-types=<X>Spec,<X>Status`,
`--shared-types-source=…/resource.go`) and `mockgen` for `Repo` and `<X>Plugin` (the plugin
mock only if the resource has a plugin — global read-only resources don't).
Ref: [resource/storage/block-storage/v1/backend/kubernetes/generate.go](../../../resource/storage/block-storage/v1/backend/kubernetes/generate.go).

### 4.4 Register in the Makefile slice list — `framework/backend/kubernetes/Makefile`
Add `"$(REPO_ROOT)/resource/<group>/<resource>/v1/backend/kubernetes" \` to the
`generate-crds` loop. **See the §3 trap.** (Already-scaffolded resources may already be in
the list — check before adding.)

### 4.5 Generate and inspect
Run `(cd resource && go generate ./...)` and `make generate-api` (or the `-ctzd` variants).
Confirm the generated files and the CRD validations per §3. Do not proceed until the CRD
reflects the spec.

### 4.6 Conversion — `resource/<group>/<resource>/v1/backend/kubernetes/conversion.go`
`<X>FromCR(obj client.Object) (*<x>dom.<X>, error)` and
`<X>ToCR(x *<x>dom.<X>) (client.Object, error)`. Use `commonbackend.ResourceStateFromCR`
(never a raw cast) and the error template `"<resource> <name>: <description>: %w"`.
Ref: block-storage / workspace `conversion.go`.

### 4.7 Plugin interface — `resource/<group>/<resource>/v1/backend/kubernetes/plugin.go`
`type <X>Plugin interface { … }`. Methods follow the resource's spec operations: `Create`
and `Delete` at minimum; add resource-specific verbs (block-storage adds
`IncreaseSize(ctx, *<x>dom.<X>) error`). Skip entirely for read-only/global resources.
Ref: [resource/storage/block-storage/v1/backend/kubernetes/plugin.go](../../../resource/storage/block-storage/v1/backend/kubernetes/plugin.go).

### 4.8 Plugin handler — `resource/<group>/<resource>/v1/backend/kubernetes/plugin_handler.go`
The reconciliation state machine. Embed `frameworkbackend.GenericPluginHandler`, hold
`repo` + `plugin`. Implement `HandleReconcile` and the `isX*`/`wantX*` predicates.
**Structural symmetry is mandatory** (CONVENTIONS §8): the `isXPending` predicate must treat
nil status as pending and only mark deleting when `DeletedAt == nil`. block-storage is the
authoritative template:
[resource/storage/block-storage/v1/backend/kubernetes/plugin_handler.go](../../../resource/storage/block-storage/v1/backend/kubernetes/plugin_handler.go).
Match the handler's complexity to the resource's spec operations: for a **create/delete-only**
resource (spec `operations: [list, get, put, delete]` with no extra verbs — e.g. `image`),
the simpler **workspace** handler
([resource/workspace/v1/backend/kubernetes/plugin_handler.go](../../../resource/workspace/v1/backend/kubernetes/plugin_handler.go))
is the better template; use block-storage's multi-state machine only when the resource has an
extra mutating operation (block-storage's size increase).

### 4.9 Controller — `resource/<group>/<resource>/v1/backend/kubernetes/controller.go`
`NewController(ctrlClient, dynClient, plugin, opts...)` wiring a `RepoAdapter`, the plugin
handler, and `frameworkcontroller.NewGenericController`.
Ref: [resource/storage/block-storage/v1/backend/kubernetes/controller.go](../../../resource/storage/block-storage/v1/backend/kubernetes/controller.go).

### 4.10 REST — `resource/<group>/<resource>/v1/frontend/rest/`
- Always add `converter.go`: `<X>FromAPI`, `<X>ToAPI`/`<X>ToAPIWithVerb`,
  `<X>IteratorToAPI`, plus an identity type and list-param helper.
- **If your resource is a new member of an existing group**, implement its
  `List/Get/CreateOrUpdate/Delete` methods on the **group owner's** `handler.go` and add a
  reader/writer field to that handler struct. (E.g. the storage handler already has Image
  methods as `501` stubs — replace them and add `ImageReader`/`ImageWriter`.) The handler
  signatures encode the scope: tenant-scoped resources take `(tenant)`, workspace-scoped take
  `(tenant, workspace)`.
- **If your resource is a new group**, add `handler.go` implementing the group's
  `ServerInterface` (mirror block-storage's handler).
Refs: storage handler above; sibling-converter pattern in
[resource/storage/storage-sku/v1/frontend/rest/converter.go](../../../resource/storage/storage-sku/v1/frontend/rest/converter.go).

### 4.11 Gateway wiring — `gateway/cmd/regionalapiserver.go` (or `globalapiserver.go`)
Create `k8sadapter.NewReaderAdapter`/`NewWriterAdapter` for the resource (GVR + `FromCR`/
`ToCR`) and either add them to the existing group handler struct, or register a new
`Sdk<group>api.HandlerWithOptions(&<group>rest.Handler{…}, …BaseURL: "/providers/seca.<group>")`
block. Regional binary for tenant/workspace/network-scoped; global binary for global.
Ref: the storage block in [gateway/cmd/regionalapiserver.go](../../../gateway/cmd/regionalapiserver.go).

### 4.12 Dummy plugin — `csp/dummy/pkg/plugin/<resource>.go` + `csp/dummy/cmd/main.go`
- New file `csp/dummy/pkg/plugin/<resource>.go`: a `type <X> struct{ logger }`, `New<X>`, and
  the `<X>Plugin` methods, each delegating to a `simulate<X>` helper. Add a `simulate<X>` to
  [csp/dummy/pkg/plugin/simulate.go](../../../csp/dummy/pkg/plugin/simulate.go) (mirror
  `simulateBS`: stamp an annotation with an expiry, return `ErrStillProcessing` until it
  elapses). Ref: [csp/dummy/pkg/plugin/block_storage.go](../../../csp/dummy/pkg/plugin/block_storage.go).
- In [csp/dummy/cmd/main.go](../../../csp/dummy/cmd/main.go): add the `<x>k8s` import,
  `utilruntime.Must(<x>k8s.AddToScheme(scheme))` in `init()`, instantiate
  `<x>Plugin := dummyplugin.New<X>(…)`, and `controllerSet.Add(<x>k8s.NewController(mgr.GetClient(), dynClient, <x>Plugin, controllerOpts...))`.

### 4.13 Deployment & RBAC (permissions)
Add **two rules** (`<plural>` and `<plural>/status`) for the resource's API group to every
relevant ClusterRole. CRDs install automatically from `chart/crd/`. Update at least:
- [csp/dummy/deploy/clusterrole.yaml](../../../csp/dummy/deploy/clusterrole.yaml) (delegator)
- [test/e2e/deploy/delegator/clusterrole.yaml](../../../test/e2e/deploy/delegator/clusterrole.yaml)
- [test/e2e/deploy/gateway-regional/clusterrole.yaml](../../../test/e2e/deploy/gateway-regional/clusterrole.yaml)
  (gateway reads; split read/write verbs as that file does)

> ⚠️ These ClusterRoles drift easily — the dummy delegator role currently lists
> `block-storages` and `workspaces` but **not** `networks`, even though `main.go` registers
> the network controller. Add your resource to *all* roles where its peers appear; don't
> copy the gap.

### 4.14 Tests
Follow existing conventions; add new scenarios you judge worth covering for this resource
(e.g. block-storage tests its size-increase path — cover your resource's distinctive verbs).
- **Unit** (per slice, `backend/kubernetes/*_test.go`): `conversion_test.go`,
  `plugin_handler_test.go`, `controller_test.go`, and an envtest (`<resource>_envtest_test.go`
  + `setup_envtest_test.go`). The plugin mock is generated by the `generate.go` mockgen line.
- **Integration** (`csp/dummy/test/integration/<resource>_test.go`): build tag
  `//go:build integration`; create via the repo adapter; poll with
  `wait.PollUntilContextTimeout(ctx, pollInterval, timeout, …)` for the expected
  `ResourceState`. Wire the repo in `main_test.go`. **Examples are inline fixtures here — no
  separate examples folder.** Ref:
  [csp/dummy/test/integration/blockstorage_test.go](../../../csp/dummy/test/integration/blockstorage_test.go).
- **E2E** (`test/e2e/test/integration/{delegator,gateway-regional}/<resource>_test.go`, or
  `gateway-global` for global resources): mirror the block-storage e2e tests.

### 4.15 Documentation (avoid doc rot)
- [README.md](../../../README.md) — only if the change is user-facing (it lists layout, not
  every resource; usually no edit needed).
- Per-folder READMEs you touched — e.g. [csp/dummy/README.md](../../../csp/dummy/README.md),
  [test/e2e/README.md](../../../test/e2e/README.md). Check each folder you modify for a
  README and update it if the resource list/behavior it documents changed.
- [doc/CODEGEN.md](../../../doc/CODEGEN.md) / [doc/PLUGINS.md](../../../doc/PLUGINS.md) — only
  if you changed the generation pipeline or plugin contract (you normally don't).

## 5. Scope, group, and region — quick reminders

- **Scope comes from the spec `hierarchy:`** (§1.3 table). It is not a free choice.
- **REST is per-API-group**, not per-resource: extend the group owner's handler; only create
  a new handler/gateway block for a brand-new group.
- **Region is special**: `resource/region/v1/` (no group dir), bare group `v1.secapi.cloud`,
  `scope=Cluster`, no plugin/controller, served by the **global** gateway.

## 6. Conventions (doc/CONVENTIONS.md — non-negotiable)

- **Conversion naming**: `XFromCR`/`XToCR`, `XFromAPI`/`XToAPI` — never `Map`/`Domain`/`CR`
  as infix tokens.
- **Initialisms** always fully capitalised in hand-written names: `API CR CIDR GB HTTP ID
  IOPS IP IPv4 IPv6 SKU URL` (`PublicIP`, `NetworkSKU`, not `PublicIp`/`NetworkSku`).
  Generated `…Spec`/`…Status` lowercase tails (e.g. `NicSpec`) are an accepted residual.
- **Typed-short variables/receivers** (`bs`, `ws`, `n`, `r`, `sku`); never shadow an import
  alias (`domain`, `resource`).
- **Structural symmetry**: parallel operations share helpers, names, and the error template
  `"<resource> <name>: <description>: %w"`. Use `commonbackend.ResourceStateFromCR`, not raw
  casts. Match block-storage's `isXPending` predicate exactly.
- **Doc comment** on every exported symbol, beginning with its name; no package-name stutter.

## 7. Verify (last step only)

```bash
make pre-commit-ctzd     # generate-api-verify + test + lint + gofmt + vuln + gosec, in Docker
make pre-merge-ctzd      # the above + rebase/workspace verification
```

`-ctzd` runs the target inside the tools container, so **Docker is the only required
toolchain**. There is no extra Makefile wiring beyond the slice list (§4.4). If Docker or
the toolchain is unavailable, **stop, warn the user, and mark the resource UNVERIFIED** —
state plainly that the checks did not run.

## 8. Commit policy

Do **not** auto-commit. If a commit is unavoidable, use a separate branch and a single-line
Conventional Commit (e.g. `feat(storage/image): implement image vertical`). No agent
attribution, no co-author trailer.

## Definition of done

- [ ] Name/plural/scope/group resolved from the spec; only-missing steps identified.
- [ ] `domain.go`, `resource.go`, `generate.go` present and correct.
- [ ] Slice added to the `framework/backend/kubernetes/Makefile` `generate-crds` loop.
- [ ] Generation run; `zz_generated_*` and `chart/crd/<group>…_<plural>.yaml` present **with
      the spec's validations**.
- [ ] `conversion.go`, `plugin.go`, `plugin_handler.go`, `controller.go` present (skip
      plugin/controller for global read-only resources).
- [ ] REST converter (+ handler methods on the group owner or a new handler) implemented.
- [ ] Gateway wired (regional or global).
- [ ] Dummy plugin file + `main.go` registration done.
- [ ] RBAC rules added to dummy and e2e ClusterRoles (`<plural>` + `<plural>/status`).
- [ ] Unit + integration (+ e2e) tests added, following existing conventions.
- [ ] Touched-folder READMEs / docs updated.
- [ ] `make pre-commit-ctzd` and `make pre-merge-ctzd` pass — or work is marked UNVERIFIED.
- [ ] No auto-commit (or: separate branch + one-line conventional commit).
